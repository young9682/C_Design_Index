# 实验六： 银行排队调度模拟系统
# 一、实验目的
1. 掌握随机数生成与随机模拟算法，实现客户到达时间、业务时长随机模拟；
2. 学会使用结构体存储客户、窗口业务数据，理解批量复合数据组织方式；
3. 运用数组模拟队列，完成多服务窗口排队仿真，掌握先进先出队列逻辑；
4. 熟练文本文件读写操作，实现仿真统计数据持久化写入日志文件；
5. 搭建循环仿真流程，完成等待时长、队列长度、业务量等多维度数据统计，理解离散事件模拟思路。
# 二、涉及知识点
随机数应用、结构体定义与数组存储、队列模拟、循环仿真、文件读写、数值统计。
# 三、问题描述及功能要求
- 编写 C 语言程序模拟银行多窗口全天排队业务流程，功能要求如下：
- 1. 随机生成客户数据：随机生成全天客户到达时刻、每位客户业务办理耗时；
- 2. 多窗口排队机制：设置多个服务窗口，先到先排队，客户到达后自动分配至当前人数最少窗口排队，每队最大排队人数为10人，17:00后不再接收新客户加入队列，最终每天各窗口的服务人数习题相同或相近；
- 3. 全天（8：00-17：00，20：00结束一切服务，强制关门）业务仿真：按时间顺序循环推进仿真，依次完成客户排队、窗口办理、客户离开全流程；
- 4. 仿真数据统计： 
- - 统计所有客户平均等待时长； 
- - 记录仿真过程中出现的最大排队长度； 
- - 分别统计每个窗口全天接待客户总数量；
5. 日志记录：将本次仿真的全部统计结果写入本地日志文件，便于后续查看；
6. 控制台输出仿真过程关键信息与最终汇总统计数据。
