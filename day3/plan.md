# 一、实验名称：智能共享充电宝租借柜系统

# 二、实验目的
1. 综合运用数组或结构体来管理多个具有不同状态的实体对象。
2. 掌握基于用户输入和系统当前状态进行复杂分支判断与流程控制。
3. 模拟现实世界中“租借–归还–计费”的完整业务闭环逻辑。
4. 提升将实际问题抽象为程序模型并进行模块化设计的能力。

# 三、实验内容
## 设计一个“智能共享充电宝租借柜”的管理程序。该租借柜共有 N 个槽位（充电宝仓位），每个槽位初始放置一个充电宝。充电宝柜通过一个触屏控制器与用户交互。系统核心状态与规则：
### 充电宝状态：每个充电宝有且仅有三种状态：
- 空闲：在槽位内，可供租借。
- 租借中：已被用户借出，不在柜内。
- 充电中：已被归还，但电量低于50%，正在柜内充电，不可被租借。
### 用户操作：用户可通过控制器选择两种操作：
- 租借：用户选择“租借”后，系统需从所有状态为“空闲”的充电宝中，随机分配一个给用户。若没有空闲的充电宝，则提示“暂无可用电宝”。租借成功后，该充电宝状态变为“租借中”，并记录租借开始时间。
- 归还：用户选择“归还”时，需输入要归还的充电宝编号。系统校验该编号的充电宝状态是否为“租借中”。若是，则完成归还，并根据租借时长计算费用（计费规则见下），同时根据归还时的电量更新其状态：
  - 若电量 ≥ 50%，状态直接变为 空闲；同时充电，每秒电量增加 0.25%，直到 100% 电量，并开始模拟充电计时，每秒更新一次电量状态。
  - 若电量 < 50%，状态变为 充电中；同时充电，每秒电量增加 0.25%，直到 100% 电量；并开始模拟充电计时，每秒更新一次电量状态。
  - 在任何情况下，若电量 ‌>= 50%‌，状态直接变为 ‌空闲。若电量 ‌< 50%‌，状态变为 ‌充电中
### 充电模拟:
- 处于“充电中”的充电宝，在系统中每经过一次完整的“时间推进”模拟（例如，每次系统主循环或一个特定的时间更新函数调用），其电量增加 20%。当电量达到或超过 100% 时，状态自动变回“空闲”。
### 系统初始化：
- 程序启动时，所有 N 个充电宝状态为 空闲，电量 100%。
### 信息展示：
- 能实时显示所有充电宝的编号、当前状态、电量（若在柜内）。
### 租借流程：
- 用户选择租借：系统查找可用的（空闲）充电宝。若有，则随机选择一个，变更其状态，记录租借时间，并提示用户租借成功（显示充电宝编号）。若无，给出明确提示。
- 归还与计费流程：用户选择归还，并输入充电宝编号。系统验证编号有效性及状态是否为租借中。
- 若验证通过，则：
1. 计算租借时长和费用。
2. 随机生成一个归还电量（例如20%–90%之间的随机数）。
3. 根据归还电量，决定充电宝下一状态（空闲或充电中）。
4. 显示归还成功、租借时长、应付费用以及充电宝后续状态（如：“充电宝X已归还，租借2.5小时，费用4元，电量不足，正在充电中。”）。
5. 若验证失败（编号不存在或状态非租借中），给出相应错误提示。
